# Презентация дипломного проекта
## Складской учёт «СКЛАД v1.0»

---

## Тема

**Почему выбрана эта тема?**

- Личная мотивация: необходимость автоматизации учёта товаров на складе
- Актуальность: большинство складов в SMB-сегменте используют Excel или бумажный учёт
- Практическая ценность: возможность масштабирования для реального использования
- Ограниченность существующих решений: дорогие ERP-системы или слишком простые таблицы

---

## Аналоги и конкуренты

| Решение | Особенности | Недостатки |
|---------|-------------|------------|
| 1C:Управление торговлей | Полнофункциональное | Высокая стоимость, сложность |
| МойСклад | Облачный сервис | Подписка, зависимость от интернета |
| Excel/Google Таблицы | Бесплатно | Нет автоматизации, риск ошибок |
| retailCRM | Для интернет-магазинов | Не для склада |

**Отличие нашего решения:** Простота, бесплатность, открытый код, возможность доработки под свои нужды

---

## Какую задачу решает приложение

**Функционал:**

- Учёт товаров с привязкой к организации
- Категоризация товаров
- Контроль остатков и минимальных порогов
- Визуальная статистика (остатки, стоимость)
- Фильтрация по категориям
- Админ-панель для управления

**Бизнес-задачи:**

- Знать, какие товары заканчиваются
- Быстро оценить стоимость остатков
- Разделять данные между организациями

---

## Целевая аудитория

- **Малый бизнес** (ИП, небольшие ООО)
- **Склады розничной торговли**
- **Небольшие производства**
- **Стартапы** с простым учётом

**Портрет пользователя:** Предприниматель без IT-образования, которому нужно простое решение для учёта товаров

## Технологии и инструменты

### Backend
- **Python 3.x** — популярный, лёгкий в изучении
- **Django 4.x** — полнофункциональный фреймворк, "батарейки в комплекте"
- **SQLite** — встроенная БД, не требует настройки

### Frontend
- **HTML5 + CSS3** — стандартная вёрстка
- **Bootstrap 5** — готовые компоненты, адаптивность
- **Bootstrap Icons** — иконки

### Инструменты
- **VS Code** — редактор кода
- **Git** — контроль версий
- **Django Admin** — готовая панель управления

---

## Почему выбраны именно они

| Технология | Причина выбора |
|------------|----------------|
| Django | MVC-архитектура, ORM, админка из коробки, большое сообщество |
| Python | Простой синтаксис, много библиотек, востребованность |
| SQLite | Для разработки/тестирования — не нужно устанавливать сервер БД |
| Bootstrap | Не нужно писать CSS с нуля, мобильная адаптация |

**Альтернативы рассматривались:** Flask (менше готовых функций), React (сложнее для новичка), PHP ( менее востребован)

## Трудности и решения

| Трудность | Решение |
|-----------|---------|
| Как считать статистику? | Использование Django ORM aggregate (Count, Sum) вместо цикла |
| Защита маршрутов | Проверка авторизации и организации в каждом view |
| Выбор организации | Использование session для хранения выбранной организации |
| Визуализация статусов | Использование @property в модели Product |
| Адаптивность | Bootstrap 5 сетка + медиа-запросы |

## Заключение

Проект "СКЛАД" решает реальную задачу учёта товаров с применением современных технологий. Простота и понятность кода делают его хорошей базой для дальнейшего развития и демонстрирует владение веб-разработкой на Django.
